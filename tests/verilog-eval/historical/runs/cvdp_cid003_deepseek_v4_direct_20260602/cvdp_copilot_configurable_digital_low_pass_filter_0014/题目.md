# cvdp_copilot_configurable_digital_low_pass_filter_0014

Category: cid003
Difficulty: easy

## Prompt
Implement a complete SystemVerilog module for `fsm_linear_reg` module, a finite state machine (FSM) for linear regression computations. It processes input data using configurable weights and biases to produce filtered results. The FSM operates in three states: `IDLE`, `COMPUTE`, and `DONE`. When triggered by a start signal, the module computes and outputs the filtered results, signaling completion with a `done` flag.

## Behavioral Description

The FSM operates in three states: **`IDLE`**, **`COMPUTE`**, and **`DONE`**.

- **IDLE**:  
  - Waits for `start` signal.  
  - Outputs are held at zero.

- **COMPUTE**:  
  - Computes results using combinational logic:  
    - **`result1`** = `w_in * x_in >>> 1`  
       - The size of **result1** is calculated to prevent overflow during multiplication when the inputs have maximum or minimum values.
    - **`result2`** = `b_in + (x_in >>> 2)`  
       - The size of **result2** is calculated to prevent overflow during the sum when the inputs have maximum or minimum values.
  - Updates the output registers.  
  - Transitions to `DONE` upon completion.

- **DONE**:  
  - Asserts `done` signal for 1 clock cycle.  
     - Even if the `start` input is always `1`, the `done` output will only be `1` for one cycle during the state transitions from `IDLE` to `COMPUTE` and from `COMPUTE` to `DONE`.
  - Returns to `IDLE` state to await the next `start`.

- **Default**:
   -  If, for any reason, there is a failure in the state transition and the next state becomes invalid, the FSM must return to the `IDLE` state.

---

## Reset and Latency

- **Reset**:  
  - When `reset` is high, all outputs (`result1`, `result2`, `done`) are reset to zero.  
  - FSM returns to `IDLE`.

- **Latency**:  
  - Fixed latency: 2 clock cycles from `start` asserted to changes in `result1`  and `result2`.
  - Fixed latency: 3 clock cycles from `start` asserted to assertion of `done`.
